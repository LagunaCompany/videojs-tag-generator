And finally... Your Code!

For use on the Arcadia websites, the HTML should be copied from the box below, and pasted into an ECMC template where you want the video to appear:

<video id="{{videoTagCtrl.video.id}}" class="video-js {{videoTagCtrl.video.skinClass}}" data-setup='{{videoTagCtrl.video.dataSetup}}' data-vjs-skin-src="{{videoTagCtrl.videoJsSkins[videoTagCtrl.video.skinClass]}}" controls autoplay loop preload='{{videoTagCtrl.video.preload}}' width='{{videoTagCtrl.video.width}}' height='{{videoTagCtrl.video.height}}' poster='{{videoTagCtrl.video.poster}}' data-tracked-video-name='{{videoTagCtrl.video.trackedVideoName}}'>
<source src="{{value}}" type="video/{{key}}" />
<div class="vjs-no-js"><a href="{{videoTagCtrl.video.sources.mp4}}" title="Download video"><span>Your browser cannot play this video, please click here to download it instead.</span><img src="{{videoTagCtrl.video.poster}}" alt="Video poster" /></a></div>
<script type="text/template" class="vjs-end-screen-content-template">
{{tinymceModel}}
</script>
</video>

That's it! The necessary JavaScript and CSS will be loaded along with the video. If it looks as though this isn't happening, please create a Lighthouse ticket and assign to the Digital IT Production Team.

If the video is intended for a third-party website, the code in the following box must also be included in the head tag of the page. The website's administrator can assist with this. Note that certain features, such as fullscreen videos, end screen content, custom buttons and Omniture tracking should not be configured if the video is to be used on a third-party website.

<link href="{{videoTagCtrl.videoJsSkins[videoTagCtrl.video.skinClass]}}" rel="stylesheet">
<script src="//vjs.zencdn.net/4.12/video.js"></script>
